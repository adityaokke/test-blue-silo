Full-Stack Development Challenge
Objective
Build a web-based Helpdesk Ticket Maintenance System (inspired by Jira) that allows users to open, track, manage, and escalate support tickets (L1–L3 levels). Focus on clean, structured UI/UX using React (Vite) and Tailwind CSS, and build the backend with Express.js (TypeScript). The system must support role-based actions and a clear escalation workflow.
🧩 Tech Stack Requirements
● Frontend: React (Vite) + TypeScript + Tailwind CSS
● Backend: Express.js + TypeScript
● Testing: Jest (Frontend & Backend)
● Database: MongoDB
● Auth: Basic role-based access (JWT or session)
● API: RESTful endpoints
● Version Control: GitHub or Bitbucket (your choice, no icons needed)
👥 User Roles & Functional Requirements
🔹 Helpdesk Agent (L1)
● Create new tickets with:
○ Title
○ Description
○ Category
○ Expected Completion Date/Time
○ Priority: Low / Medium / High
● Update ticket status: New, Attending, Completed
● Escalate to L2 if unresolved
🔹 Technical Support (L2)
● View escalated tickets
● Assign Critical Value:
○ C1 (High) – System down
○ C2 (Medium) – Partial feature issue
○ C3 (Low) – Minor problem or inquiry
● Add actions & update ticket logs
● Escalate to L3 if unresolved
🔹 Advanced Support (L3)
● Handle only escalated critical (C1–C2) tickets
● Add resolution and close the ticket
🔁 Ticket Escalation Flow
L1: Create Ticket ➜ Attempt Fix ➜ Escalate if unresolved
L2: Assign Critical Value ➜ Work on Fix ➜ Escalate if unresolved
L3: Final Fix ➜ Mark as Resolved
📦 Must-Have Features
● Login with role-based permissions (L1, L2, L3)
● Ticket list (with filters: status, priority, escalation)
● Ticket detail view (description, history/logs, current status)
● Escalate with notes
● Update ticket status at each level
● Log actions taken at L2/L3
● Clean, minimal UI
● Unit tests (backend & frontend)
